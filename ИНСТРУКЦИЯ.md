# Установка и работа — раскрой из спецификации (Windows)

Цельная инструкция: установка с нуля через виртуальную среду (venv),
система контроля версий (Git) и тесты. Команды копируй целиком.

Итоговая структура:

```
C:\raskroy\
├── venv\                  ← виртуальная среда (создаётся командой)
├── raskroy.py             ← главный скрипт
├── sp_to_cutlist.py       ← конвертер
├── requirements.txt       ← библиотеки с версиями
├── README.md
├── .gitignore
├── smart_cut_app\         ← калькулятор (с GitHub)
│   ├── materials_catalog.json   ← справочник склада
│   └── leftovers_db.json        ← справочник остатков
└── tests\
    ├── test_razbor.py
    └── data\karkas_etalon.xls
```

## ШАГ 1. Установить Python

1. Открой **https://www.python.org/downloads/**
2. Скачай **Python 3.12.x**, запусти установщик.
3. **ВАЖНО:** галочка ☑ **Add python.exe to PATH** на первом экране.
4. **Install Now**.

Проверка (`Win+R` → `cmd` → Enter):
```cmd
python --version
```
Должно показать `Python 3.12.x`.

---

## ШАГ 2. Собрать папку проекта

1. Создай `C:\raskroy\`.
2. Скачай калькулятор: **https://github.com/Nivoronkov/smart_cut_app**
   → кнопка **Code** → **Download ZIP** → распакуй так, чтобы получилось
   `C:\raskroy\smart_cut_app\` (внутри сразу `main.py`, `core`, `data`).
3. Скопируй в `C:\raskroy\` файлы из моего пакета:
   `raskroy.py`, `sp_to_cutlist.py`, `requirements.txt`, `README.md`,
   `.gitignore`, и папку `tests\` целиком (с `data\karkas_etalon.xls`).

Проверка:
```cmd
dir C:\raskroy\smart_cut_app\main.py
dir C:\raskroy\raskroy.py
dir C:\raskroy\tests\test_razbor.py
```

## ШАГ 3. Создать виртуальную среду (venv) и поставить библиотеки

**Один раз:**
```cmd
cd C:\raskroy
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

После `activate` в начале строки появится `(venv)` — среда включена.
Установка через `requirements.txt` ставит библиотеки фиксированных версий
(одинаково на любой машине).

> **Каждый новый сеанс работы** начинай с активации:
> ```cmd
> cd C:\raskroy
> venv\Scripts\activate
> ```
> Если забыть — будет ошибка `No module named pandas`. Это не поломка,
> просто напоминание активировать среду.

## ШАГ 4. Завести Git (контроль версий)

Ты знаком с Git, поэтому коротко — что важно именно тут.

**Инициализация (один раз):**
```cmd
cd C:\raskroy
git init
git add .
git commit -m "Базовая версия: конвертер + калькулятор + первый тест"
```

`.gitignore` уже настроен правильно:
- **не коммитятся:** `venv/`, `__pycache__/`, рабочие `*.xls` и результаты
  `*_вход.xlsx` / `*_раскрой.xlsx`, логи;
- **коммитятся:** код, `requirements.txt`, и тестовый эталон
  `tests/data/*.xls` (он нужен для прогона тестов — для него сделано
  исключение в `.gitignore`).

**Рабочий ритм** при доработке конвертера:
```cmd
# поправил sp_to_cutlist.py, прогнал тесты (см. Шаг 6), убедился что зелено
git add sp_to_cutlist.py
git commit -m "Разбор уголков: добавил профиль Уголок"
```
Сломалось и нужно откатиться к последней рабочей версии:
```cmd
git checkout -- sp_to_cutlist.py     # отменить несохранённые правки файла
# или вернуться к конкретному коммиту, если уже закоммитил лишнее
```

Бэкап на GitHub (по желанию): создай приватный репозиторий и
`git remote add origin ...` + `git push`. Это и резервная копия, и история.

## ШАГ 5. Запуск расчёта

Положи спецификацию (`.xls` из Компаса) в `C:\raskroy\`. Затем:
```cmd
cd C:\raskroy
venv\Scripts\activate
python raskroy.py "014_1-26-10_300_Каркас.xls"
```

Скрипт: конвертирует → сверяет со складом → считает раскрой.
Рядом появятся:
- `..._вход.xlsx` — **что система поняла** (листы Детали / Материалы /
  Проверка / Отсечено). Смотри его глазами на этапе отладки.
- `..._раскрой.xlsx` — **готовая карта раскроя**.

## ШАГ 6. Тесты (защита от поломок)

После **каждой** правки конвертера прогоняй:
```cmd
cd C:\raskroy
venv\Scripts\activate
pytest
```

- **`7 passed` (зелёно)** — разбор эталонной спецификации «Каркас» не сломан,
  можно коммитить.
- **`failed` (красно)** — твоя правка что-то поломала. pytest покажет, какой
  именно инвариант разошёлся (число деталей, длина, отсечение и т.п.).
  Чини до зелёного, потом коммить.

Когда добавишь разбор новых спецификаций (с исполнениями, позициями) —
пришли мне файл, я добавлю под него отдельный тест-эталон. Так набор тестов
будет расти вместе с обработкой, и каждая новая спецификация будет
застрахована.

## Возможные проблемы

| Что видишь | Причина | Решение |
|---|---|---|
| `python не является командой` | Нет галочки PATH | Переустановить Python с Add to PATH |
| `No module named pandas` | venv не активирован | `venv\Scripts\activate` |
| `No module named pytest` | Не поставлены библиотеки | Повторить Шаг 3 |
| `ModuleNotFoundError: core` | `smart_cut_app` лежит не там | Проверить структуру (Шаг 2) |
| `Файл не найден` | Имя/путь спецификации | Имя в кавычках, проверь расположение |
| Окно не открывается | Нет PySide6 | `pip install -r requirements.txt` |

## Известные ограничения (на текущем этапе)

1. **Несколько длин одного материала** (6 м и 12 м одновременно) движок
   калькулятора пока сам не перебирает. Обходной путь — заводить как
   отдельные позиции склада. Доработка оптимизатора — позже.
2. **Импорт спецификации в окне** не реализован (см. Шаг 7).
3. **Нормализация марок стали** (`Ст3пс3-св` → `Ст3пс`) и спорные случаи
   («Сталь 10» у трубы 50х50х4) требуют твоего экспертного решения и
   справочника соответствий — сделаем при доработке конвертера.
