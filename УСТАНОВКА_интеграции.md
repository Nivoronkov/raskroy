# Интеграция: кнопка «Загрузить спецификацию» в окне калькулятора

Теперь спецификацию Компас можно загрузить прямо в окно калькулятора,
без командной строки. Весь процесс — мышкой.

## Что установить

В папке `C:\raskroy\smart_cut_app\ui\`:

1. **Заменить** файл `main_window.py` на новый (из этого пакета).
2. **Добавить** новый файл `import_specification.py` (из этого пакета).

Больше ничего не меняется. Конвертер `sp_to_cutlist.py` уже лежит в
`C:\raskroy\` — модуль импорта его находит автоматически.

> Если используешь Git: после замены сделай коммит
> `git add smart_cut_app/ui/ && git commit -m "Интеграция: импорт спецификации в GUI"`

## Как пользоваться

1. Запусти окно:
   ```cmd
   cd C:\raskroy\smart_cut_app
   ..\venv\Scripts\activate
   python main.py
   ```
2. Меню **Файл → Загрузить спецификацию (Компас .xls)**.
3. Выбери файл спецификации (.xls).
4. Детали автоматически попадут на вкладку «Детали».
   - номер исполнения и доработка — в колонке «Примечание»
     (оператор увидит их на карте раскроя);
   - если каких-то материалов нет в справочнике склада — появится
     предупреждение со списком: их надо добавить.
5. Если были предупреждения — открой вкладку **«Справочник материалов»**,
   добавь недостающие материалы (укажи длины хлыстов).
6. Вкладка **«Параметры расчёта»** → кнопка **«Рассчитать»**.
7. Результат — на вкладке **«Результаты»**: карта раскроя с цветными
   полосами, в каждой — длина детали и примечание (исполнение).

## Важно про справочник склада

Расчёт в окне берёт материалы **только из справочника**. Деталь, для
которой материала в справочнике нет, в расчёт не попадёт. Поэтому порядок
такой: загрузил спецификацию → увидел, чего не хватает → добавил в
справочник → рассчитал.

Это та самая логика «материал распознан, но его нет на складе — добавь»,
теперь прямо в окне.

## Что осталось от командного режима

Командный `raskroy.py` никуда не делся — он удобен для быстрой пакетной
проверки многих спецификаций и для отладки (файл «Вход» со светофором).
Окно — для рабочего расчёта с визуальной картой. Оба используют один и
тот же конвертер, результат идентичный.

## Если что-то не так

| Сообщение | Причина | Решение |
|---|---|---|
| «Не найден файл конвертера sp_to_cutlist.py» | Конвертер не в C:\raskroy\ | Положи sp_to_cutlist.py на уровень выше smart_cut_app |
| «Не найдено деталей линейного проката» | В спецификации только листы/крепёж | Это норма — линейного проката в файле нет |
| Материалы не попадают в расчёт | Их нет в справочнике | Добавь на вкладке «Справочник материалов» |
